# Урок 5 — индексы и explain

where задаёт логическое условие отбора строк.
Индекс — это структура данных, которая позволяет базе
найти эти строки быстрее, не просматривая всю таблицу.

Индекс:
- ускоряет select / join / order by
- замедляет insert / update / delete
- занимает дополнительное место

explain нужен, чтобы понять:
- использует ли postgres индекс
- или делает seq scan (полный просмотр таблицы)

Важно:
- на маленьких таблицах postgres часто выбирает seq scan,
  даже если индекс есть (это нормально)
- explain показывает план, а не выполняет запрос
